# 1. Purpose

This document defines the philosophy of the ARK Workspace.

Unlike traditional dashboards, the Workspace is designed around the operator rather than predefined reports.

Its objective is to provide a calm, flexible and task-oriented working environment.

---

# 2. Philosophy

The system must not decide what is important today.

The operator decides.

Every trading day may begin with a different question.

Examples:

- What is the current Bitcoin price?
- Are all orders finalized?
- Which campaigns are still active?
- How much free capital is available?
- Did any customer hit Stop Loss?
- Which customers are still waiting?
- Which exchange requires attention?

The Workspace must adapt to today's priorities.

---

# 3. Dashboard vs Workspace

Traditional dashboards assume that important information is fixed.

ARK rejects this assumption.

Important information changes depending on:

- Market conditions
- Active campaigns
- Operator objectives
- Customer status
- Exchange status

Therefore ATH uses a Workspace instead of a Dashboard.

---

# 4. Workspace Principles

The Workspace must be:

- Calm
- Minimal
- Flexible
- Operator-driven
- Context-aware

The Workspace should never overwhelm the operator with unnecessary information.

---

# 5. Widgets

Everything visible inside the Workspace is a Widget.

Examples:

- BTC Price
- Exchange Status
- Active Campaigns
- Pending Orders
- Free Capital
- Customer Overview
- Risk Alerts
- Trade Console

Widgets are independent building blocks.

---

# 6. Personalization

Each operator may arrange the Workspace differently.

Possible operations include:

- Add Widget
- Remove Widget
- Resize Widget
- Reorder Widget
- Collapse Widget

The system should remember the preferred layout.

---

# 7. Empty Space is Valuable

An empty screen is not wasted space.

It creates focus.

The Workspace should only contain information intentionally chosen by the operator.

Silence is a feature.

---

# 8. Operator Mindset

Operators do not begin every day with the same objective.

One day the priority is execution.

Another day the priority is monitoring.

Another day the priority is analysis.

The Workspace must support changing priorities without requiring configuration changes by developers.

---

# 9. Future Vision

The Workspace philosophy is not limited to Trade Hub.

It is expected to become a common interaction model across future ARK products.

---

# 10. Design Principles

The Workspace follows these principles:

- User controls the interface.
- Information appears by intention.
- Context defines importance.
- Simplicity over density.
- Focus over decoration.
- Calm over noise.
